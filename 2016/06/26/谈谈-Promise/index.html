一些概念

为什么要用 Promise?

首先必须要知道的是 Javascript 是单线程的，并且在 Javascript 中存在很多异步的过程。
举个例子，人们在医院里体检，分别要在各个不同的科室中进行检查，每一间科室不会等待某一位患者完成之前的检查并拒绝所有其他的用户，正确的行为是只要有能够检查的，就先让能检查的进行检查。

什么是 Promise

传统的 Javascript 的异步编程方法是回调 (callback)，意即在过程完成后调用 callback 函数处理返回结果，使用传统的 callback 方法，有可能会导致可怕的回调地狱(其实三角形看起来挺好看的吖:P)

而使用 Promise 能够让我们用一种并行的方式编写异步代码
关于 Promise) 的相关内容就不再赘述，接下来简单的讲一讲相关的概念:

  • Promise
    用于表示还未知的值
  • Deferred
    用于表示还未完成的工作
    当你理解了 Promise 之后你会觉得上面这两个概念其实就已经能够概括 Promise 了
    具体一点:
    1. Promise 是一个初始未知值的占位符
    2. Deferred 是产生未知值的计算过程
    3. Deferred 有一个 Promise 对象用来代表未来的结果
    4. 调用者可以 resolve/reject Deferred
    5. Promise 本身可以被分配给多个消费者，并且每个消费者都会独立观察 Promise 的结果
    6. resolver 和 deferred 可以被分配给任意数量的生产者，并且 Promise 能够被第一个 resolve 处理
      用连续的语言形容一下，大概是这样子的：
      我们有一个异步的过程 A，如果使用 Promise 的方式来处理这个异步过程，那么 A 会产生一个 Deferred - B，同时 B 中包含这一个 Promise - C，这个 C 代表的是 A 可能产生的结果，如果我们将 B 分配给另一个需要 A 产生数据的过程 D，那么当 B 的结果返回时，D 的就能够 C。
